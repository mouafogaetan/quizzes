Fonctions et Procédures en Algorithmie

Condensé des notions essentielles avec exemples et exercices

Introduction

Les fonctions et procédures permettent de structurer un algorithme en le divisant en sous-parties réutilisables. Elles offrent plusieurs avantages :

  • Réduction de la taille des algorithmes
  • Facilitation de la lecture et compréhension
  • Simplification de la correction d'erreurs
  • Réutilisation de code

1. Les Procédures

Définition

Une procédure est un sous-algorithme qui effectue une tâche spécifique sans retourner de valeur.

Syntaxe :
Procédure NomProcédure(paramètre1: type1, ..., paramètreN: typeN)
    Déclaration des variables locales
Début
    // Instructions de la procédure
FinProcédure
Exemple :

Procédure qui affiche la somme de deux nombres :

Procédure addition(a: Réel, b: Réel)
    Var S: Réel
Début
    S ← a + b
    Afficher("La somme est :", S)
FinProcédure
Appel d'une procédure :
Algorithme ExempleAppel
    Var x, y: Réel
Début
    Afficher("Entrez deux nombres")
    Lire(x, y)
    addition(x, y)  // Appel de la procédure
Fin

2. Les Fonctions

Définition

Une fonction est un sous-programme qui effectue un calcul et retourne un résultat.

Syntaxe :
Fonction NomFonction(paramètre1: type1, ..., paramètreN: typeN): TypeRetour
    Déclaration des variables locales
Début
    // Instructions de la fonction
    Retourner résultat
FinFonction
Exemple :

Fonction qui calcule et retourne la somme de deux nombres :

Fonction addition(a: Réel, b: Réel): Réel
    Var S: Réel
Début
    S ← a + b
    Retourner S
FinFonction
Appel d'une fonction :
Algorithme ExempleAppel
    Var x, y, resultat: Réel
Début
    Afficher("Entrez deux nombres")
    Lire(x, y)
    resultat ← addition(x, y)  // Appel de la fonction
    Afficher("La somme est :", resultat)
Fin

3. Paramètres et Variables

Concepts importants
Variables locales vs globales :
  • Variable locale : Déclarée et utilisable uniquement dans la fonction/procédure où elle est définie
  • Variable globale : Déclarée dans l'algorithme principal et utilisable partout
Règles pour les paramètres :
  • Les arguments doivent apparaître dans l'ordre choisi pendant la déclaration
  • Les arguments doivent être du même type que les paramètres correspondants

4. Application au calcul de moyenne

Problème initial

Créer un algorithme pour calculer la moyenne des élèves en utilisant fonctions et procédures.

Exercice 1 : Fonction de calcul de moyenne

Écrire une fonction qui calcule la moyenne pondérée à partir des notes et coefficients.

La formule de la moyenne pondérée est :

$$moyenne = \frac{\sum_{i=1}^{n} (note_i \times coefficient_i)}{\sum_{i=1}^{n} coefficient_i}$$ 
Fonction calculMoyenne(notes: Tableau de Réel, coefficients: Tableau de Réel): Réel
    Var i, sommeNotes, sommeCoeffs: Réel
Début
    sommeNotes ← 0
    sommeCoeffs ← 0
    
    Pour i de 0 à taille(notes)-1 faire
        sommeNotes ← sommeNotes + (notes[i] * coefficients[i])
        sommeCoeffs ← sommeCoeffs + coefficients[i]
    FinPour
    
    Retourner sommeNotes / sommeCoeffs
FinFonction
Exercice 2 : Procédure de saisie des notes

Écrire une procédure qui permet à un utilisateur de saisir les notes pour une matière.

Procédure saisieNotes(matiere: Chaîne, ref notes: Tableau de Réel)
    Var i, nbNotes: Entier
Début
    Afficher("Combien de notes pour ", matiere, "?")
    Lire(nbNotes)
    
    Redimensionner(notes, nbNotes)
    
    Pour i de 0 à nbNotes-1 faire
        Afficher("Saisir note ", i+1, " pour ", matiere)
        Lire(notes[i])
    FinPour
FinProcédure
Exercice 3 : Algorithme complet

Écrire un algorithme complet qui utilise les fonctions et procédures précédentes pour calculer la moyenne d'un élève.

Algorithme CalculMoyenneEleve
    Var matieres: Tableau de Chaîne ← ["Maths", "Physique", "Français"]
    Var coeffs: Tableau de Réel ← [4, 3, 2]
    Var notesMaths, notesPhysique, notesFrancais: Tableau de Réel
    Var moyenneMaths, moyennePhysique, moyenneFrancais, moyenneGenerale: Réel
    
    // Saisie des notes pour chaque matière
    saisieNotes("Maths", notesMaths)
    saisieNotes("Physique", notesPhysique)
    saisieNotes("Français", notesFrancais)
    
    // Calcul des moyennes par matière
    moyenneMaths ← calculMoyenne(notesMaths, [3, 2, 1]) // Exemple de coefficients
    moyennePhysique ← calculMoyenne(notesPhysique, [2, 2, 1])
    moyenneFrancais ← calculMoyenne(notesFrancais, [1, 1, 1])
    
    // Calcul de la moyenne générale
    moyenneGenerale ← calculMoyenne([moyenneMaths, moyennePhysique, moyenneFrancais], coeffs)
    
    // Affichage des résultats
    Afficher("Moyenne en Maths:", moyenneMaths)
    Afficher("Moyenne en Physique:", moyennePhysique)
    Afficher("Moyenne en Français:", moyenneFrancais)
    Afficher("Moyenne générale:", moyenneGenerale)
Fin

5. Différences clés entre fonctions et procédures

Comparaison
Aspect Fonction Procédure
Valeur de retour Retourne une valeur Ne retourne pas de valeur
Utilisation Dans une expression (affectation, calcul) Instruction indépendante
Mot-clé Fonction ... : TypeRetour Procédure
Instruction de retour Retourner valeur Pas de retour
Conclusion

Les fonctions et procédures sont des outils essentiels en algorithmie pour :

  • Structurer le code
  • Éviter la répétition
  • Faciliter la maintenance
  • Améliorer la lisibilité

Pour les utiliser correctement, il faut respecter l'ordre, le type et le nombre des paramètres à chaque appel, cohérents avec leur déclaration.
