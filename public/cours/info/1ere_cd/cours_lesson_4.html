Structures Algorithmiques - Résumé

Ce document présente un condensé des notions essentielles sur les structures algorithmiques, avec des exemples et exercices corrigés.

1. Structures Itératives (Boucles)

Structure TANT QUE

Exécute une action tant qu'une condition est vraie.

// Syntaxe
TANT QUE condition FAIRE
  action;
FIN TANT QUE

Exercice: Afficher les nombres de 1 à 5 avec une boucle TANT QUE.

Variables i: Entier
Début
  i ← 1;
  TANT QUE i ≤ 5 FAIRE
    AFFICHER(i);
    i ← i + 1;
  FIN TANT QUE
Fin
Structure RÉPÉTER JUSQU'À

Exécute une action au moins une fois, puis répète jusqu'à ce qu'une condition soit vraie.

// Syntaxe
RÉPÉTER
  action;
JUSQU'À condition

Exercice: Demander un nombre jusqu'à ce qu'il soit positif.

Variables n: Entier
Début
  RÉPÉTER
    AFFICHER("Entrez un nombre positif: ");
    LIRE(n);
  JUSQU'À n > 0
Fin
Structure POUR

Exécute une action un nombre déterminé de fois.

// Syntaxe
POUR variable ALLANT DE début À fin [PAS valeur] FAIRE
  action;
FIN POUR

Exercice: Calculer la somme des nombres de 1 à 10.

Variables i, somme: Entier
Début
  somme ← 0;
  POUR i ALLANT DE 1 À 10 FAIRE
    somme ← somme + i;
  FIN POUR
  AFFICHER("La somme est: ", somme);
Fin

2. Structures Conditionnelles

Structure SI ALORS SINON (complète)

Exécute une action si une condition est vraie, une autre action si elle est fausse.

// Syntaxe
SI condition ALORS
  action1;
SINON
  action2;
FIN SI

Exercice: Vérifier si un nombre est pair ou impair.

Variables n: Entier
Début
  AFFICHER("Entrez un nombre: ");
  LIRE(n);
  SI n MOD 2 = 0 ALORS
    AFFICHER("Le nombre est pair");
  SINON
    AFFICHER("Le nombre est impair");
  FIN SI
Fin
Structure SI ALORS (simple)

Exécute une action seulement si une condition est vraie.

// Syntaxe
SI condition ALORS
  action;
FIN SI

Exercice: Afficher un message si la note est supérieure à 10.

Variables note: Réel
Début
  AFFICHER("Entrez votre note: ");
  LIRE(note);
  SI note > 10 ALORS
    AFFICHER("Vous avez réussi");
  FIN SI
Fin

3. Structures de Données

Enregistrements (Structures)

Regroupement de données de types différents sous un même nom.

// Déclaration d'un type Personne
Type
  Structure TPersonne
    nom: Chaîne
    âge: Entier
  FinStruct

Exercice: Créer une variable de type Personne et afficher ses informations.

Variables p: TPersonne
Début
  p.nom ← "Dupont";
  p.âge ← 25;
  AFFICHER("Nom: ", p.nom, ", Âge: ", p.âge);
Fin
Tableaux

Collection d'éléments de même type, accessibles par un indice.

// Déclaration d'un tableau de 10 entiers
Variables notes: Tableau[10] de Entier

Exercice: Calculer la moyenne des notes d'un tableau.

Variables notes: Tableau[5] de Réel
      i: Entier
      somme, moyenne: Réel
Début
  // Saisie des notes
  POUR i ALLANT DE 0 À 4 FAIRE
    AFFICHER("Entrez la note ", i+1, ": ");
    LIRE(notes[i]);
  FIN POUR

  // Calcul de la moyenne
  somme ← 0;
  POUR i ALLANT DE 0 À 4 FAIRE
    somme ← somme + notes[i];
  FIN POUR
  moyenne ← somme / 5;
  AFFICHER("La moyenne est: ", moyenne);
Fin
Listes Chaînées

Structure de données où chaque élément pointe vers le suivant.

4. Exercices de Synthèse

Exercice 1: Écrire un algorithme qui trouve le plus grand élément dans un tableau d'entiers.

Variables tab: Tableau[10] de Entier
      i, max: Entier
Début
  // Remplissage du tableau (omis pour la brièveté)
  max ← tab[0];
  POUR i ALLANT DE 1 À 9 FAIRE
    SI tab[i] > max ALORS
      max ← tab[i];
    FIN SI
  FIN POUR
  AFFICHER("Le plus grand élément est: ", max);
Fin

Exercice 2: Écrire un algorithme qui inverse les éléments d'un tableau.

Variables tab: Tableau[10] de Entier
      i, temp: Entier
Début
  POUR i ALLANT DE 0 À 4 FAIRE
    temp ← tab[i];
    tab[i] ← tab[9-i];
    tab[9-i] ← temp;
  FIN POUR
Fin

Exercice 3: Écrire un algorithme qui calcule la factorielle d'un nombre entier positif.

Variables n, i, fact: Entier
Début
  AFFICHER("Entrez un nombre entier positif: ");
  LIRE(n);
  fact ← 1;
  POUR i ALLANT DE 1 À n FAIRE
    fact ← fact * i;
  FIN POUR
  AFFICHER("La factorielle de ", n, " est: ", fact);
Fin
