Adressage Réseau

Concepts fondamentaux et bonnes pratiques

Introduction à l'adressage réseau

Un réseau informatique doit permettre à tout équipement connecté de communiquer avec n'importe quel autre équipement. Pour garantir une identification sans ambiguïté, trois éléments essentiels sont nécessaires :

  1. Le nom de la machine distante (qui)
  2. Son adresse réseau (où)
  3. La route à suivre pour l'atteindre (comment)

L'adresse IP (Internet Protocol version 4) identifie de manière unique l'emplacement d'un hôte sur le réseau. Elle est représentée sous la forme d'un nombre de 32 bits, généralement exprimé en quatre nombres décimaux séparés par des points (par exemple, 192.168.1.1).

Problème pratique

Dans un bureau, trois ordinateurs sont connectés en réseau, mais l'un d'eux ne parvient pas à voir les autres machines sur le réseau. Quelle pourrait être la cause de ce problème ?

Réponse : Vérifier la configuration de l'adressage IP des machines.

Question 2 : Type de réseau

Quel type de réseau les collaborateurs ont-ils probablement mis en place ?

Réponse : Un LAN (réseau local) privé.

Question 3 : Adresses IP et connexion Internet

Si ce réseau local est connecté à Internet, les mêmes adresses IP pourront-elles être utilisées ?

Réponse : Non, car les adresses privées ne sont pas routables sur Internet.

Types d'adresses IP

On distingue principalement deux types d'adresses IP :

1. Adresses privées

Ces adresses peuvent être librement utilisées par les administrateurs de réseaux locaux, à condition qu'elles ne soient pas routées sur Internet. Elles sont réservées aux réseaux privés.

Plages d'adresses privées :

  • 10.0.0.0 à 10.255.255.255
  • 172.16.0.0 à 172.31.255.255
  • 192.168.0.0 à 192.168.255.255

2. Adresses publiques

Ces adresses sont attribuées par l'ICANN (Internet Corporation for Assigned Names and Numbers) et garantissent l'unicité mondiale. La distribution est assurée par les FAI (Fournisseurs d'Accès à Internet).

L'ICANN délègue la gestion des adresses à des organisations régionales (RIR) :

  • APNIC (Asie-Pacifique)
  • ARIN (Amérique du Nord)
  • RIPE NCC (Europe, Moyen-Orient et Asie centrale)
  • AfriNIC (Afrique)
  • LACNIC (Amérique latine et Caraïbes)

Outre ces deux catégories principales, on trouve également :

  • Adresses Unicast : Identifient un équipement unique (ex: ordinateur, serveur)
  • Adresses Multicast : Permettent d'envoyer des messages à un groupe d'équipements
  • Adresses Broadcast : Permettent la diffusion à tous les équipements d'un sous-réseau
Question 4 : Reconnaissance du réseau

Lorsqu'un réseau local est connecté à Internet via un FAI, comment peut-on reconnaître l'adresse de son propre réseau dans le grand réseau Internet ?

Réponse : En analysant la structure de l'adresse IP et en utilisant le masque de sous-réseau.

Structure d'une adresse IPv4

Une adresse IPv4 est un nombre de 32 bits, généralement représenté sous forme de quatre octets séparés par des points. Chaque octet peut prendre une valeur entre 0 et 255.

L'adresse IP est composée de deux parties :

  1. Identifiant réseau (NetID) : Identifie le réseau
  2. Identifiant hôte (HostID) : Identifie l'équipement sur le réseau

Représentation schématique :

4 octets (32 bits)
Partie réseau (NetID) Partie hôte (HostID)

Note : La partie réseau peut occuper 1, 2 ou 3 octets selon la classe de l'adresse.

Classes d'adresses IP

Les adresses IPv4 sont divisées en classes pour faciliter leur attribution et leur gestion :

Classe Masque par défaut Plage d'adresses Nombre de réseaux Nombre d'hôtes
A 255.0.0.0 1.0.0.0 à 126.0.0.0 126 $(2^7 - 2)$ 16 777 214
B 255.255.0.0 128.0.0.0 à 191.255.0.0 16 384 $(2^{14})$ 65 534
C 255.255.255.0 192.0.0.0 à 223.255.255.0 2 097 152 $(2^{21})$ 254
D Non défini 224.0.0.0 à 239.255.255.0 Réservé au multicast
E Non défini 240.0.0.0 à 255.255.255.0 Usage expérimental

La classe d'une adresse IP est déterminée par les premiers bits de l'adresse :

  • Classe A : Premier bit à 0 (0xxxxxxx)
  • Classe B : Deux premiers bits à 10 (10xxxxxx)
  • Classe C : Trois premiers bits à 110 (110xxxxx)
  • Classe D : Quatre premiers bits à 1110 (1110xxxx)
  • Classe E : Quatre premiers bits à 1111 (1111xxxx)

Adresses IP réservées

Certaines adresses IP ont une signification particulière :

  • HostID = 0 : Désigne le réseau lui-même (ex: 192.168.1.0)
  • HostID = 255 : Adresse de diffusion dirigée (ex: 192.168.1.255)
  • 0.0.0.0 : Hôte courant sur le réseau local
  • 255.255.255.255 : Diffusion limitée au réseau local
  • 127.0.0.1 : Adresse de boucle locale (loopback)
Question 5 : Nombre total d'adresses IPv4

Combien d'adresses IPv4 différentes peut-on théoriquement avoir ? Est-ce suffisant pour tous les équipements connectés à Internet dans le monde ?

Réponse : On peut avoir $2^{32} = 4 294 967 296$ adresses IPv4 différentes. Ce nombre est insuffisant face à la croissance exponentielle des équipements connectés à Internet.

Conclusion

Le protocole IPv4, standardisé en septembre 1981, reste largement utilisé aujourd'hui. Cependant, face à l'explosion du nombre d'appareils connectés à Internet, ses limitations sont devenues évidentes.

Pour pallier cette pénurie d'adresses, IPv6 (Internet Protocol version 6) a été développé. Cette nouvelle version utilise des adresses de 128 bits, offrant un espace d'adressage considérablement élargi. Une adresse IPv6 est représentée par huit groupes de quatre chiffres hexadécimaux séparés par des deux-points (ex: 2001:0db8:85a3:0000:0000:8a2e:0370:7334).

Exercices pratiques

  1. Convertir les adresses IP suivantes en binaire :
    • 145.32.59.24
    • 200.42.129.16
  2. Déterminer la classe des adresses IP suivantes :
    • 10000000.00001010.11011000.00100111
    • 11101101.10000011.00001110.01011111
    • 01001010.00011011.10001111.00010010
  3. Pour chaque adresse, souligner la partie demandée :
    • Partie réseau : 1.102.45.177
    • Partie hôte : 196.22.177.13
    • Partie réseau : 133.156.55.102
  4. Pour une machine configurée avec l'adresse IP 192.168.1.1 et un masque de réseau 255.255.255.0 :
    • Donner l'adresse du réseau et l'adresse de diffusion
    • Répéter l'exercice avec l'adresse IP 172.26.17.100 et le masque 255.255.240.0
    • Répéter l'exercice avec l'adresse IP 193.48.57.163 et le masque 255.255.255.224
