Architecture Logicielle des Systèmes Informatiques

Comprendre les fondements du fonctionnement des ordinateurs

Introduction

Lorsqu'on observe un ordinateur en fonctionnement, une question fondamentale se pose : comment une machine constituée de composants électroniques peut-elle comprendre et exécuter nos instructions ? La réponse réside dans l'architecture logicielle qui donne vie au matériel informatique.

Question 1 :

Qu'est-ce qui permet de donner des ordres à un ordinateur qui est à la base formé uniquement de composants électroniques ?

Réponse : Les programmes informatiques

Schéma montrant la place du système d'exploitation dans l'architecture logicielle

I. Concepts Fondamentaux

Un système informatique comprend deux parties essentielles :

  • Le matériel (Hardware) : composants physiques de l'ordinateur
  • Le logiciel (Software) : ensemble des programmes qui permettent au système d'accomplir ses tâches
Définitions importantes :

Information : Élément de connaissance susceptible d'être transformé pour être traité par un système informatique.

Donnée : Information représentée sous une forme conventionnelle permettant un traitement automatique.

Langage informatique : Ensemble de caractères, symboles, mots-clés et règles permettant de formuler des instructions pour un ordinateur.

Instruction : Ordre donné à un processeur.

Programme : Séquence d'instructions exécutable par l'ordinateur.

Logiciel : Ensemble de programmes coordonnés.

Types de logiciels :

Progiciel : Ensemble complet de programmes conçus pour différents utilisateurs dans un domaine d'application spécifique, généralement commercialisé avec une documentation.

Freeware (Gratuiciel) : Logiciel entièrement gratuit, pouvant être copié librement.

Logiciel libre (Open Source) : Application livrée avec son code source, permettant sa modification et son adaptation. Attention : libre ne signifie pas nécessairement gratuit.

Logiciel propriétaire : Logiciel dont l'utilisation est soumise à des restrictions définies par son éditeur.

Ludiciel : Logiciel de divertissement (jeux vidéo).

Didacticiel : Logiciel destiné à l'enseignement et à l'apprentissage.

Question 2 :

Tous les programmes sont-ils fournis avec l'ordinateur lors de l'achat ?

Réponse : Non, seuls certains logiciels de base sont préinstallés.

Question 3 :

Comment appelle-t-on le petit programme intégré par les constructeurs lors de la fabrication ?

Réponse : Le BIOS

II. Le BIOS (Basic Input Output System)

Le BIOS, ou Système de Gestion de Base des Entrées/Sorties, est un programme fondamental installé par le constructeur lors de la fabrication de l'ordinateur.

Interface typique d'un BIOS

Rôles principaux du BIOS :
  • Vérifier l'état du matériel au démarrage
  • Permettre l'installation des premiers programmes
  • Exécuter le POST (Power On Self Test) lors de la mise sous tension

Le POST est une procédure de test des différents composants matériels. Si un problème est détecté, des messages d'erreur sont affichés à l'écran.

Le BIOS est stocké dans une mémoire de type ROM appelée CMOS (Complementary Metal-Oxide Semiconductor). Cette mémoire nécessite une alimentation constante par une pile pour conserver les informations de configuration du système (heure, taille de la mémoire vive, caractéristiques des disques, etc.).

Question 4 :

Comment appelle-t-on le premier programme qu'on installe sur un ordinateur ?

Réponse : Le système d'exploitation (SE)

III. Le Système d'Exploitation
1. Définition

Un système d'exploitation (SE) est un ensemble de programmes qui assurent le bon fonctionnement de l'ordinateur. Il sert d'interface entre l'utilisateur et le matériel, facilitant l'interaction entre ces deux éléments.

Architecture typique d'un noyau de système d'exploitation

Exemples de systèmes d'exploitation :
  • Windows
  • macOS
  • Linux
  • Unix
  • Android
  • iOS
2. Fonctions principales d'un système d'exploitation
  • Gestion du processeur
  • Gestion de la mémoire
  • Gestion des entrées/sorties
  • Gestion des programmes
  • Gestion des utilisateurs
3. Classification des systèmes d'exploitation

Selon la gestion des tâches :

  • Systèmes monotâche : exécutent une seule tâche à la fois
  • Systèmes multitâche : permettent l'exécution simultanée de plusieurs tâches

Selon la gestion des postes :

  • Systèmes monoposte : conçus pour un seul ordinateur à la fois
  • Systèmes multipostes : permettent à plusieurs ordinateurs de partager des ressources

Selon la gestion des utilisateurs :

  • Systèmes mono-utilisateur : ne permettent qu'à un seul utilisateur de travailler à la fois
  • Systèmes multi-utilisateurs : autorisent plusieurs utilisateurs à travailler simultanément
Question 5 :

Comment appelle-t-on les logiciels utilisés pour exécuter des tâches spécifiques avec un ordinateur ?

Réponse : Les logiciels d'application

IV. Les Logiciels d'Application

On distingue deux grandes catégories de logiciels :

  1. Les logiciels de base (systèmes d'exploitation)
  2. Les logiciels d'application

Un logiciel d'application est un ensemble de programmes spécialement conçus pour exécuter une tâche précise sur l'ordinateur. Ils sont classés par catégories selon leur domaine d'application.

Exemples de logiciels d'application :
  • Traitement de texte (Word, Google Docs)
  • Tableurs (Excel, Sheets)
  • Logiciels de conception graphique (Photoshop, GIMP)
  • Navigateurs web (Chrome, Firefox)
  • Logiciels de communication (Zoom, Skype)
  • Ludiciels (jeux vidéo)
  • Didacticiels (logiciels éducatifs)
Les utilitaires

Parmi les logiciels d'application, on trouve les utilitaires - des programmes qui exécutent des opérations annexes aux fonctions principales du système d'exploitation.

Les pilotes (drivers) sont un type particulier d'utilitaires qui permettent au système d'exploitation de communiquer avec les périphériques matériels (imprimante, carte graphique, etc.).

Conclusion

L'architecture logicielle d'un système informatique est une structure complexe et hiérarchisée qui permet à des composants matériels de comprendre et d'exécuter nos instructions. Du BIOS au système d'exploitation, puis aux applications spécialisées, chaque couche logicielle joue un rôle essentiel dans le fonctionnement global de l'ordinateur.

Comprendre cette architecture est fondamental pour saisir comment les ordinateurs traitent l'information et exécutent les tâches qui leur sont confiées.
