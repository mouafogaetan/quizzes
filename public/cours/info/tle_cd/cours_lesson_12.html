UE 13 : Commandes Réseaux et Simulation d'un Réseau

Cours de 2 heures pour les classes de Terminale C/D

Compétences visées

À la fin de cette leçon, l'élève doit être capable de :

  • Utiliser les commandes réseaux ipconfig et ping
  • Simuler un réseau informatique

Situation problème

Dans le cadre des travaux pratiques au laboratoire informatique, il vous est demandé de concevoir un réseau local constitué de 4 postes de travail dans le but de prédire son comportement avant son fonctionnement. Votre conception doit être représentée sur support papier et support numérique.

Consignes :
  1. Énumérer les équipements matériels ainsi que les supports d'interconnexion et de transmission
  2. Proposer un exemple de logiciel permettant de produire le support numérique
  3. Préciser la commande qui permettra de tester les connexions de tous les postes de travail
  4. Donner la commande qui affiche les paramètres réseaux d'un ordinateur

Introduction

De nos jours, la mise en place d'un réseau informatique nécessite de faire des simulations avant le lancement en production. Cette bonne pratique est indispensable car elle permet d'avoir une idée sur le comportement futur du réseau et d'éviter de nombreuses erreurs.

Ainsi, les tests de connexion et la vérification des paramètres réseaux des équipements peuvent être effectués à l'aide des commandes réseaux. Dans la suite de notre leçon, nous décrirons quelques commandes réseaux et comment simuler un réseau informatique.

I. Les commandes réseaux

Une commande réseau représente une instruction donnée au système d'exploitation pour exécuter une action dans le domaine des réseaux informatiques. Chaque commande répond à une syntaxe spécifique et doit être entrée dans l'invite de commande (pour les systèmes Windows) ou dans un Terminal (pour les systèmes UNIX) pour s'exécuter.

1. La commande Ipconfig

Cette commande affiche ou rafraîchit la configuration réseau d'un ordinateur.

Sa syntaxe est la suivante : Ipconfig [option]. Pour l'utiliser, il suffit d'ouvrir l'invite de commande, de saisir la commande et de valider en tapant la touche Entrée.

Exemple : Ipconfig /all

Avec l'option /all, cette commande affiche les informations détaillées sur la configuration réseau d'un ordinateur :

  • Nom de l'hôte
  • Adresse IP en cours
  • Masque de sous-réseau
  • Passerelle par défaut
  • Adresses MAC des cartes réseaux
  • État des serveurs DHCP, etc.

L'exécution de la commande Ipconfig sans option affiche l'adresse IP, le masque réseau et la passerelle par défaut d'une machine.

2. La commande Ping

La commande Ping permet de tester la connectivité d'une machine à travers un réseau. Pour vérifier la disponibilité d'un ordinateur, la commande Ping envoie des requêtes (4 par défaut) vers l'adresse IP de cet ordinateur grâce au protocole ICMP (Internet Control Message Protocol).

Sa syntaxe est : ping [adresse_machine] où adresse_machine représente l'adresse IP, le nom de la machine ou un nom de domaine.

Exemples :

  • ping www.google.fr : teste l'accessibilité du site web Google
  • ping www.facebook.com : teste l'accessibilité du site web Facebook
  • ping /? : affiche toutes les informations sur l'utilisation de la commande Ping

Le résultat de la commande Ping permet de connaître :

  • L'adresse IP correspondant au nom de la machine distante
  • La durée de vie des paquets ou TTL (Time To Live)
  • Le temps de propagation des paquets (en millisecondes)
  • Le nombre de paquets perdus

II. La simulation réseau

La simulation réseau représente une technique permettant de créer et de configurer un réseau informatique virtuel à l'aide d'un logiciel spécialisé (appelé simulateur).

L'utilisation d'un simulateur réseau présente plusieurs avantages :

  • Connaissance de l'ensemble des équipements réseaux disponibles dans le simulateur
  • Possibilité de visualiser le fonctionnement de tous les équipements à chaque échange d'informations
  • Possibilité d'animer le déplacement d'un message d'une machine à une autre

Il existe plusieurs logiciels de simulation réseau parmi lesquels :

  • Cisco Packet Tracer
  • GNS3
  • Netsim
  • NS2

Conclusion

Il est important de simuler un réseau informatique avant de le réaliser car cela permet d'avoir une idée sur son comportement et d'éviter de nombreuses erreurs. De plus, cette simulation fait appel aux commandes réseaux pour effectuer les tests de connectivité.

Exercices pratiques

Exercice 1 : Utilisation des commandes Ping et Ipconfig

Lors d'une séance de TP au laboratoire informatique, utilisez les commandes réseaux dans l'invite de commandes pour afficher les paramètres réseaux des postes et effectuer les tests de connexion.

  1. Accéder à l'invite de commande de votre ordinateur :
    • Cliquer sur le « Menu Démarrer »
    • Saisir la commande « cmd » dans la zone de recherche
    • Valider avec la touche Entrée
  2. Pour avoir plus de détails sur la commande Ping, entrer la commande ping /?
  3. Pour avoir plus de détails sur la commande Ipconfig, entrer la commande ipconfig /?
  4. Activer votre connexion Internet et entrer les commandes suivantes :
    • ipconfig
    • ipconfig /all

    Question : Quelles différences pouvez-vous observer entre les résultats obtenus avec ces deux commandes ?

  5. Tester la connectivité avec les commandes :
    • ping www.yahoo.fr
    • ping www.google.fr

    Questions :

    • Décrire l'état de la connexion dans chacun des résultats obtenus
    • Les temps de réponse des paquets sont-ils identiques ? Pourquoi ?

Exercice 2 : Simulation d'un réseau local avec Cisco Packet Tracer

Simulez un réseau local dont la topologie est présentée ci-dessous :

Topologie réseau à simuler avec adresse réseau : 192.168.1.0 et masque : 255.255.255.0

Consignes détaillées :

  1. Démarrer l'application Cisco Packet Tracer
  2. Identifier les différentes parties de l'interface
  3. Ajouter 4 ordinateurs dans l'espace de travail (PC0, PC1, PC2, PC3)
  4. Ajouter un commutateur (Switch)
  5. Connecter les postes au switch à l'aide de câbles droits
  6. Attribuer les adresses IP aux ordinateurs :
    • PC0 : 192.168.1.1
    • PC1 : 192.168.1.2
    • PC2 : 192.168.1.3
    • PC3 : 192.168.1.4
    Masque de sous-réseau : 255.255.255.0
  7. Consulter les configurations réseaux des ordinateurs et tester les connexions avec les commandes ipconfig et ping
  8. Visualiser les échanges d'informations en mode simulation
  9. Enregistrer votre travail et fermer l'application
