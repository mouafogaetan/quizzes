Introduction aux espaces vectoriels

Les espaces vectoriels constituent l'une des structures fondamentales des mathématiques modernes. Ils permettent de développer des théorèmes généraux applicables à divers ensembles mathématiques tels que les vecteurs du plan ou de l'espace, les fonctions, les suites, les polynômes, etc.

Les applications des espaces vectoriels sont nombreuses et variées, incluant :

  • Les méthodes de calcul numérique
  • Le graphisme, particulièrement la 3D
  • La résolution d'équations aux dérivées partielles (comme l'équation de la chaleur)
  • L'analyse de systèmes vibratoires
Motivation et importance

La notion d'espace vectoriel est essentielle car elle fournit un cadre unifié pour traiter divers problèmes mathématiques. En identifiant une structure commune à différents ensembles, nous pouvons développer des outils puissants qui s'appliquent à tous ces contextes.

Exemple concret :

L'ensemble des polynômes de degré inférieur ou égal à 2, noté $\mathbb{R}_2[X]$, possède une structure d'espace vectoriel. Cela signifie que nous pouvons additionner deux polynômes de degré ≤ 2 et multiplier un tel polynôme par un réel, tout en restant dans $\mathbb{R}_2[X]$.

Pré-requis et rappels

Avant d'aborder les espaces vectoriels, assurez-vous de maîtriser :

  1. La résolution de systèmes d'équations linéaires
  2. Les combinaisons linéaires de vecteurs
  3. Les notions de base sur les matrices et déterminants
Exercice de rappel :

Résoudre dans $\mathbb{R}^3$ les systèmes :

(S₁) $\begin{cases} x + y + 3z = 0 \\ 2x + y - z = 0 \\ -x + 2y - z = 0 \end{cases}$

(S₂) $\begin{cases} x + y + 3z = 0 \\ 2x + y - z = 0 \end{cases}$

Solution :

Pour (S₁) : L'unique solution est $(0; 0; 0)$

Pour (S₂) : Les solutions sont de la forme $(4z; -7z; z)$ où $z \in \mathbb{R}$

Objectifs pédagogiques
  • Reconnaître un sous-espace vectoriel d'un espace vectoriel
  • Déterminer si une famille de vecteurs est libre ou génératrice
  • Trouver une base dans un espace vectoriel
  • Montrer que deux sous-espaces vectoriels sont supplémentaires
  • Déterminer si une application entre espaces vectoriels est linéaire
  • Écrire la matrice d'une application linéaire par rapport à des bases données
  • Déterminer le noyau et l'image d'une application linéaire
Sous-espaces vectoriels
Définition :

Une partie $H$ non vide d'un espace vectoriel $E$ sur $\mathbb{R}$ est un sous-espace vectoriel de $E$ si :

  1. $H$ est non vide
  2. Pour tous $\vec{u}, \vec{v} \in H$, $\vec{u} + \vec{v} \in H$ (stabilité par addition)
  3. Pour tout $\vec{u} \in H$ et tout $\lambda \in \mathbb{R}$, $\lambda \cdot \vec{u} \in H$ (stabilité par multiplication scalaire)
Propriétés importantes :
  • L'intersection de deux sous-espaces vectoriels est un sous-espace vectoriel
  • La réunion de deux sous-espaces vectoriels n'est généralement pas un sous-espace vectoriel
  • Toute combinaison linéaire d'éléments d'un sous-espace vectoriel appartient à ce sous-espace
Exemple :

L'ensemble $A = \{ax^2 + bx + c \mid a, b, c \in \mathbb{R}\}$ des polynômes de degré inférieur ou égal à 2 est un sous-espace vectoriel de l'ensemble de tous les polynômes.

Pour le démontrer, on vérifie que :

  1. Le polynôme nul appartient à $A$
  2. La somme de deux polynômes de degré ≤ 2 est un polynôme de degré ≤ 2
  3. Le produit d'un polynôme de degré ≤ 2 par un scalaire est un polynôme de degré ≤ 2
Familles libres et génératrices
Famille libre :

Une famille de vecteurs $\{\vec{v_1}, \vec{v_2}, ..., \vec{v_n}\}$ d'un espace vectoriel $E$ est dite libre (ou les vecteurs sont linéairement indépendants) si la seule combinaison linéaire nulle de ces vecteurs est celle dont tous les coefficients sont nuls :

$\alpha_1\vec{v_1} + \alpha_2\vec{v_2} + ... + \alpha_n\vec{v_n} = \vec{0} \Rightarrow \alpha_1 = \alpha_2 = ... = \alpha_n = 0$

Famille génératrice :

Une famille de vecteurs $\{\vec{v_1}, \vec{v_2}, ..., \vec{v_n}\}$ est dite génératrice de $E$ si tout vecteur de $E$ peut s'exprimer comme combinaison linéaire de ces vecteurs :

Pour tout $\vec{v} \in E$, il existe des scalaires $\alpha_1, \alpha_2, ..., \alpha_n$ tels que $\vec{v} = \alpha_1\vec{v_1} + \alpha_2\vec{v_2} + ... + \alpha_n\vec{v_n}$

Bases et dimension
Base d'un espace vectoriel :

Une famille de vecteurs constitue une base de $E$ si elle est à la fois libre et génératrice.

Dimension :

Si un espace vectoriel $E$ (différent de $\{\vec{0}\}$) admet une base finie, alors toutes les bases de $E$ ont le même nombre d'éléments. Ce nombre est appelé dimension de $E$, noté $\dim(E)$.

Par convention, la dimension de l'espace $\{\vec{0}\}$ est 0.

Exemple : Base canonique de $\mathbb{R}^3$

La famille $\{(1, 0, 0), (0, 1, 0), (0, 0, 1)\}$ forme une base de $\mathbb{R}^3$, appelée base canonique. Ainsi, $\dim(\mathbb{R}^3) = 3$.

Propriétés importantes en dimension finie :
  • Dans $\mathbb{R}^2$, deux vecteurs $\vec{u}(a, b)$ et $\vec{v}(a', b')$ forment une base si et seulement si $\det(\vec{u}, \vec{v}) = \begin{vmatrix} a & a' \\ b & b' \end{vmatrix} \neq 0$
  • Tout système libre de $n$ vecteurs dans un espace de dimension $n$ est une base
  • Toute famille génératrice de $n$ vecteurs dans un espace de dimension $n$ est une base
Applications linéaires
Définition :

Une application $f : E \to F$ entre deux espaces vectoriels réels est dite linéaire si elle satisfait les deux conditions suivantes :

  1. $f(\vec{u} + \vec{v}) = f(\vec{u}) + f(\vec{v})$ pour tous $\vec{u}, \vec{v} \in E$
  2. $f(\lambda \cdot \vec{u}) = \lambda \cdot f(\vec{u})$ pour tout $\vec{u} \in E$ et tout $\lambda \in \mathbb{R}$

Ces deux conditions peuvent être condensées en une seule :

$f(\alpha\vec{u} + \beta\vec{v}) = \alpha f(\vec{u}) + \beta f(\vec{v})$ pour tous $\vec{u}, \vec{v} \in E$ et tous $\alpha, \beta \in \mathbb{R}$

Exemple :

L'application $f : \mathbb{R}^2 \to \mathbb{R}^2$ définie par $f(x, y) = (x - 2y, 3x - 5y)$ est linéaire.

Pour le vérifier, on montre que pour tous $\vec{u}, \vec{v} \in \mathbb{R}^2$ et tout $\lambda \in \mathbb{R}$ :

  1. $f(\vec{u} + \vec{v}) = f(\vec{u}) + f(\vec{v})$
  2. $f(\lambda \cdot \vec{u}) = \lambda \cdot f(\vec{u})$
Vocabulaire spécifique :
  • Forme linéaire : Application linéaire d'un espace vectoriel vers $\mathbb{R}$
  • Endomorphisme : Application linéaire d'un espace vectoriel dans lui-même
  • Isomorphisme : Application linéaire bijective entre deux espaces vectoriels
  • Automorphisme : Endomorphisme bijectif (isomorphisme d'un espace dans lui-même)
Noyau et image d'une application linéaire
Noyau :

Le noyau d'une application linéaire $f : E \to F$, noté $\ker(f)$, est l'ensemble des vecteurs de $E$ dont l'image par $f$ est le vecteur nul de $F$ :

$\ker(f) = \{\vec{u} \in E \mid f(\vec{u}) = \vec{0}_F\}$

Le noyau est un sous-espace vectoriel de $E$.

Image :

L'image d'une application linéaire $f : E \to F$, notée $\operatorname{Im}(f)$, est l'ensemble des vecteurs de $F$ qui admettent un antécédent par $f$ dans $E$ :

$\operatorname{Im}(f) = \{\vec{v} \in F \mid \exists \vec{u} \in E, f(\vec{u}) = \vec{v}\}$

L'image est un sous-espace vectoriel de $F$.

Théorème du rang :

Si $E$ est de dimension finie, alors :

$\dim(\ker(f)) + \dim(\operatorname{Im}(f)) = \dim(E)$

Matrices d'applications linéaires
Matrice associée à une application linéaire :

Soit $f : E \to F$ une application linéaire entre espaces vectoriels de dimensions finies. Soient $B = \{\vec{e_1}, \vec{e_2}, ..., \vec{e_p}\}$ une base de $E$ et $B' = \{\vec{e'_1}, \vec{e'_2}, ..., \vec{e'_n}\}$ une base de $F$.

La matrice de $f$ relativement aux bases $B$ et $B'$ est la matrice $M$ de taille $n \times p$ dont la $j$-ème colonne est constituée des coordonnées de $f(\vec{e_j})$ dans la base $B'$.

Exemple :

Soit $g : \mathbb{R}^3 \to \mathbb{R}^2$ définie par $g(x, y, z) = (2x + y - z, x - 3y + 4z)$.

La matrice de $g$ relativement aux bases canoniques est :

$M_g = \begin{pmatrix} 2 & 1 & -1 \\ 1 & -3 & 4 \end{pmatrix}$

Propriétés :
  • Une application linéaire est un isomorphisme si et seulement si sa matrice (dans des bases données) est inversible
  • La composition d'applications linéaires correspond au produit de leurs matrices
  • L'application réciproque d'un isomorphisme a pour matrice la matrice inverse
